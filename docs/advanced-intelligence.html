Architecture Overview

The advanced intelligence modules are designed to compose. Each operates independently but they share a common pattern: factory/constructor creation, event ingestion, autonomous analysis with scoring, insight generation, and state export/import.

BotMimicryDetector

src/bot-mimicry-detector.js — Detects sophisticated bots that deliberately imitate human behavioral patterns. While basic detectors flag inhuman speed or mechanical precision, mimicry detection catches bots that have learned to appear human by finding "uncanny valley" signals: behaviors that are too perfect, too consistent, or too ideal.

Six Detection Engines

EngineWhat It DetectsWeight
uncannyValleySuspiciously ideal human metrics — solve times, accuracy, inter-arrival timing that match textbook distributions too closely0.20
consistencyParadoxFake randomness — variance-of-windows and autocorrelation tests that expose scripted "jitter" masquerading as natural variation0.20
fatigueImmunityNo degradation over long sessions — humans slow down and make more mistakes; bots that mimic humans often forget to degrade0.15
templateMatchBehavioral fingerprints matching known mimicry toolkits via cosine-similarity against an operator-curated template library0.15
microPatternSub-second timing entropy analysis — timing distributions bucketed into 50 bins; low entropy reveals bot clock precision0.15
crossSessionSame source, same behavioral profile across different sessions — checks signature similarity against recent sessions0.15

Mimicry Tiers

TierScore RangeMeaning
GENUINE0–20Normal human behavior
LIKELY_HUMAN20–40Minor anomalies, probably human
SUSPICIOUS40–60Mixed signals, warrants monitoring
LIKELY_MIMICRY60–80Strong mimicry indicators
CONFIRMED_MIMICRY80–100Definitive bot mimicry detected

Usage

var createBotMimicryDetector = require("gif-captcha/src/bot-mimicry-detector");

var detector = createBotMimicryDetector({
  maxSessions: 5000,
  minEventsForAnalysis: 8,
  weights: { uncannyValley: 0.25, fatigueImmunity: 0.20 }  // override defaults
});

// Record solve events
detector.recordEvent({
  sessionId: "sess-abc",
  solveTimeMs: 2850,
  correct: true,
  timestamp: Date.now()
});

// Analyze a session
var result = detector.analyzeSession("sess-abc");
// { score: 72.3, tier: "LIKELY_MIMICRY", engines: { ... }, insights: [...] }

// Manage mimicry templates
detector.addTemplate({
  name: "selenium-puppeteer-v3",
  signature: { meanSolveMs: 2500, stddevMs: 200, accuracyRate: 0.95 }
});

// Export/import for persistence
var state = detector.exportState();
detector.importState(state);

Key Methods

recordEvent(event) ingestion

Records a CAPTCHA solve event for a session. Automatically manages LRU eviction when maxSessions or maxEventsPerSession limits are reached.

FieldTypeDescription
sessionIdstringUnique session identifier
solveTimeMsnumberTime to solve in milliseconds
correctbooleanWhether the solve was correct
timestampnumberEvent timestamp (ms since epoch)

analyzeSession(sessionId) returns object

Runs all six detection engines against the session’s events and returns a composite mimicry assessment. Requires at least minEventsForAnalysis events (default 8).

Returns: { score, tier, engines: { [name]: { score, details } }, insights: [...] }

addTemplate(template) / removeTemplate(name)

Manage the mimicry template library. Templates are behavioral signatures of known bot toolkits. The templateMatch engine compares session signatures against these via cosine similarity.

getStats() returns object

Returns aggregate statistics: total sessions analyzed, tier distribution, top mimicry suspects (sorted by score), template match counts, and engine-level breakdowns.

BotAttributionEngine

src/bot-attribution-engine.js — Attributes bot activity to specific operators and campaigns by analyzing toolchain fingerprints, infrastructure patterns, and operational signatures. While fraud-ring-detector finds human CAPTCHA farms and bot-collective-intel detects swarm coordination, this engine identifies the actor behind the bots — linking separate bot instances to the same threat actor.

8-Dimensional Fingerprinting

Each bot’s activity is compressed into an 8-dimensional vector capturing:

  1. Solve timing distribution (mean, variance, skew)
  2. Error pattern signature
  3. Session duration profile
  4. Request cadence and burst patterns
  5. Challenge selection preferences
  6. Geographic/IP cluster behavior
  7. User-agent and header fingerprint
  8. Retry and backoff strategy

Bots with similar fingerprint vectors (measured by cosine similarity) are grouped under the same operator.

Usage

var createBotAttributionEngine = require("gif-captcha/src/bot-attribution-engine");

var engine = createBotAttributionEngine({
  similarityThreshold: 0.82,   // cosine similarity for operator matching
  maxOperators: 500,
  maxBotsPerOperator: 1000
});

// Ingest bot activity
engine.ingestBotActivity({
  botId: "bot-xyz",
  solveTimeMs: 1200,
  correct: false,
  headers: { "User-Agent": "..." },
  ip: "203.0.113.42",
  timestamp: Date.now()
});

// Attribute a bot to an operator
var attr = engine.attributeBot("bot-xyz");
// { operatorId: "op-12", confidence: 0.91, matchedDimensions: [...] }

// Detect coordinated campaigns
var campaign = engine.detectCampaign({ minBots: 5, timeWindowMs: 86400000 });
// { campaignId: "...", botIds: [...], phase: "ESCALATING", confidence: 0.87 }

// Threat assessment for an operator
var threat = engine.assessThreatLevel("op-12");
// { level: "HIGH", sophistication: 7.2, botCount: 23, activeCampaigns: 2 }

// Merge operators when identified as the same actor
engine.mergeOperators("op-12", "op-34");

Key Methods

ingestBotActivity(event) ingestion

Records bot activity and automatically computes/updates the bot’s fingerprint vector. If the bot hasn’t been assigned to an operator yet, triggers operator matching.

attributeBot(botId) returns object

Returns the operator attribution for a known bot, including confidence score and which fingerprint dimensions matched most strongly.

detectCampaign(opts) returns object

Scans operator activity for coordinated campaigns. Returns campaign details including lifecycle phase (PROBING, ESCALATING, SUSTAINED, DECLINING) and confidence score.

assessThreatLevel(operatorId) scoring

Produces a threat assessment for an operator: sophistication score (0–10), bot count, campaign count, escalation trend, and recommended response.

mergeOperators(id1, id2) / getOperatorTimeline(id)

Manual operator management — merge two operator profiles when confirmed as the same actor, or retrieve the chronological activity timeline for forensic analysis.

getSummary() returns object

Overall attribution summary: total operators tracked, top operators by bot count, active campaigns, and unattributed bot percentage.

ChallengeDifficultyCurveEngine

src/challenge-difficulty-curve-engine.js — Models the relationship between challenge difficulty and solve outcomes to find and maintain the optimal difficulty sweet spot: maximum bot rejection with minimum human frustration. Autonomously tracks drift as bot capabilities evolve.

How It Works

The engine divides the difficulty spectrum into configurable buckets and tracks human pass rates and bot rejection rates at each level. It builds two curves:

  • Human pass curve — Monotonically decreasing as difficulty rises
  • Bot reject curve — Monotonically increasing as difficulty rises

The optimal zone is where the two curves create the largest gap (high bot rejection, acceptable human pass rate). The engine continuously monitors for difficulty drift — when bots adapt and the optimal zone shifts.

Usage

var ChallengeDifficultyCurveEngine = require("gif-captcha/src/challenge-difficulty-curve-engine");

var curve = new ChallengeDifficultyCurveEngine({
  bucketCount: 20,          // difficulty buckets from 0 to 1
  minSamplesPerBucket: 50,  // statistical significance threshold
  driftWindowMs: 86400000   // check drift over 24h
});

// Record outcomes at various difficulty levels
curve.recordSample({ difficulty: 0.45, isBot: false, solved: true });
curve.recordSample({ difficulty: 0.45, isBot: true,  solved: false });

// Find the optimal difficulty zone
var optimal = curve.findOptimalDifficulty();
// { difficulty: 0.52, humanPassRate: 0.84, botRejectRate: 0.93, confidence: 0.88 }

// Predict outcomes at a specific difficulty
var prediction = curve.predictOutcome(0.6);
// { humanPassRate: 0.71, botRejectRate: 0.97, abandonmentRate: 0.12 }

// Detect difficulty drift
var drift = curve.detectDrift();
// { drifted: true, direction: "harder", magnitude: 0.08, recommendation: "..." }

// Detect bot adaptation
var adaptation = curve.detectBotAdaptation();
// { adapted: true, previousRejectRate: 0.95, currentRejectRate: 0.78, atDifficulty: 0.5 }

// Get abandonment curve for UX planning
var abandon = curve.getAbandonmentCurve();
// [{ difficulty: 0.1, rate: 0.02 }, { difficulty: 0.2, rate: 0.05 }, ...]

// Overall calibration health
var health = curve.computeHealth();
// { score: 78, issues: ["Bot adaptation detected at mid-range"], recommendations: [...] }

Key Methods

recordSample(sample) ingestion

Records a solve attempt with its difficulty level, bot/human classification, and outcome. Samples are bucketed by difficulty for curve construction.

findOptimalDifficulty() returns object

Computes the difficulty level that maximizes botRejectRate - (1 - humanPassRate). Requires sufficient samples per bucket for statistical confidence.

detectDrift() / detectBotAdaptation()

detectDrift compares current optimal difficulty against historical snapshots. detectBotAdaptation checks whether bot rejection rates are declining at previously effective difficulty levels.

generateRecommendations() returns array

Produces autonomous difficulty adjustment recommendations based on current curves, drift analysis, and bot adaptation signals.

classifyDifficulty(value) / getBuckets() / getSummary()

Utility methods for inspecting internal state: classify a difficulty value into its bucket, retrieve all bucket data, or get an aggregate summary.

ChallengeEcosystemHealthEngine

src/challenge-ecosystem-health.js — Models the entire challenge pool as a biological ecosystem where challenges are “species,” bots are “predators,” and human solvers are “pollinators.” Provides holistic health metrics that go beyond individual challenge monitoring.

Ecosystem Dimensions

DimensionBiological AnalogyWhat It Measures
biodiversityShannon indexDiversity of challenge types and difficulty levels in active rotation
predatorPreyLotka-VolterraBot-vs-challenge dynamics — whether bots are “consuming” challenges faster than new ones emerge
carryingCapacityK in logistic growthMaximum challenge pool size sustainable given current traffic and solve patterns
extinctionRiskRed listChallenges at risk of becoming ineffective (low usage + declining solve quality)
evolutionPressureSelection pressureBot adaptation speed — how fast bots are learning to crack current challenge types
nicheAnalysisEcological nichesOver/under-served difficulty bands where challenges are clustered or absent
keystonesKeystone speciesCritical challenges that anchor the pool — their removal would cascade failures

Usage

var ChallengeEcosystemHealthEngine = require("gif-captcha/src/challenge-ecosystem-health");

var ecosystem = new ChallengeEcosystemHealthEngine({
  snapshotIntervalMs: 3600000,   // hourly snapshots
  maxSnapshots: 168              // one week of history
});

// Register challenges in the pool
ecosystem.register({ id: "gif-math-01", category: "math", difficulty: 0.5 });
ecosystem.register({ id: "gif-pattern-07", category: "pattern", difficulty: 0.7 });

// Record events (solves, failures, bot attempts)
ecosystem.recordEvent({ challengeId: "gif-math-01", isBot: false, solved: true });
ecosystem.recordEvent({ challengeId: "gif-math-01", isBot: true, solved: true }); // bot cracked it

// Run full ecosystem analysis
var health = ecosystem.analyze();
// { score: 67, biodiversity: 0.82, predatorPrey: { ... }, extinctionRisk: [...],
//   evolutionPressure: 0.45, niches: { ... }, keystones: [...], insights: [...] }

// Retire a compromised challenge
ecosystem.retire("gif-math-01");

// Render an HTML dashboard snippet
var html = ecosystem.renderDashboard();

// Historical tracking
var history = ecosystem.getHistory();
// [{ timestamp: ..., score: 72, ... }, { timestamp: ..., score: 67, ... }]

Key Methods

register(challenge) / retire(challengeId)

Add challenges to the ecosystem or retire them. Retired challenges are excluded from future analysis but preserved in historical data.

recordEvent(event) ingestion

Records a solve attempt. Events drive all ecosystem metrics — biodiversity tracking, predator-prey dynamics, extinction risk scoring, and evolution pressure measurement.

analyze() scoring

Runs the full ecosystem analysis across all seven dimensions and produces a composite health score (0–100) with autonomous insights and recommendations.

renderDashboard() returns string

Generates an HTML fragment visualizing ecosystem health suitable for embedding in admin dashboards.

ChallengeRetirementEngine

src/challenge-retirement-engine.js — Detects when challenges have been cracked by bots and proactively retires them before they become security liabilities. Tracks effectiveness decay, detects burst attacks, correlates cross-challenge compromises, and schedules graceful retirements.

Lifecycle Tiers

TierDescriptionAction
ACTIVEHealthy — strong human pass rate, low bot successContinue serving
WARNINGEarly degradation signals detectedIncrease monitoring frequency
PROBATIONSignificant effectiveness declineReduce serving weight; prepare replacement
RETIREDCompromised or exhaustedRemove from active rotation

Detection Capabilities

  • Effectiveness decay — Models bot success rate over time with configurable half-life; detects accelerating decay
  • Burst attacks — Identifies sudden automated solve spikes (high volume + high success in short windows)
  • Time-to-solve anomalies — Detects bots getting progressively faster at specific challenges
  • Cross-challenge correlation — When multiple related challenges are compromised simultaneously, flags coordinated attacks

Usage

var ChallengeRetirementEngine = require("gif-captcha/src/challenge-retirement-engine");

var retirement = new ChallengeRetirementEngine({
  decayHalfLifeMs: 604800000,  // 7-day half-life
  burstWindowMs: 60000,         // 1-minute burst detection
  burstThreshold: 20,           // 20 solves in burst window = suspicious
  warningThreshold: 0.6,        // tier thresholds
  probationThreshold: 0.4
});

// Record solve attempts
retirement.recordAttempt({
  challengeId: "gif-math-01",
  isBot: true,
  solved: true,
  solveTimeMs: 800,
  timestamp: Date.now()
});

// Analyze a specific challenge
var status = retirement.analyze("gif-math-01");
// { tier: "WARNING", effectivenessScore: 0.58, botSuccessRate: 0.42,
//   burstDetected: false, decayRate: 0.03, insights: [...] }

// Check for correlated compromises
var correlations = retirement.detectCorrelations();
// [{ challengeIds: ["gif-math-01", "gif-math-02"], correlation: 0.89, likely: "shared_vulnerability" }]

// Get the retirement log
var log = retirement.getRetirementLog();
// [{ challengeId: "gif-old-03", retiredAt: ..., reason: "effectiveness_below_threshold" }]

// Manually retire or reinstate
retirement.retire("gif-math-01", "manual");
retirement.reinstate("gif-math-01");

// Get full fleet status
var fleet = retirement.getStatus();
// { active: 45, warning: 8, probation: 3, retired: 12, insights: [...] }

Key Methods

recordAttempt(attempt) ingestion

Records a solve attempt for a challenge. Automatically updates effectiveness scoring, burst detection counters, and time-to-solve tracking.

analyze(challengeId) scoring

Runs full analysis on a challenge: effectiveness score, tier classification, decay rate, burst detection, and generated insights.

detectCorrelations() returns array

Scans for simultaneous compromise patterns across challenges. High correlation between degradation timelines suggests shared vulnerabilities or coordinated attacks.

getStatus() returns object

Fleet-wide retirement status: counts per tier, overall fleet health, autonomous insights, and a list of challenges approaching retirement.

retire(id, reason) / reinstate(id)

Manual lifecycle management. retire moves a challenge to RETIRED with a logged reason. reinstate moves it back to ACTIVE (e.g., after patching a vulnerability).

Integration Patterns

Composing the Intelligence Layer

These modules are designed to feed each other. A typical production integration:

var createMimicryDetector = require("gif-captcha/src/bot-mimicry-detector");
var createAttributionEngine = require("gif-captcha/src/bot-attribution-engine");
var ChallengeDifficultyCurveEngine = require("gif-captcha/src/challenge-difficulty-curve-engine");
var ChallengeEcosystemHealthEngine = require("gif-captcha/src/challenge-ecosystem-health");
var ChallengeRetirementEngine = require("gif-captcha/src/challenge-retirement-engine");

// Initialize
var mimicry    = createMimicryDetector();
var attribution = createAttributionEngine();
var difficulty  = new ChallengeDifficultyCurveEngine();
var ecosystem   = new ChallengeEcosystemHealthEngine();
var retirement  = new ChallengeRetirementEngine();

// On every CAPTCHA solve event:
function onSolveEvent(event) {
  // Feed all engines
  mimicry.recordEvent(event);
  difficulty.recordSample({
    difficulty: event.challengeDifficulty,
    isBot: event.isBot,
    solved: event.solved
  });
  ecosystem.recordEvent(event);
  retirement.recordAttempt(event);

  // If flagged as bot, feed attribution
  if (event.isBot) {
    attribution.ingestBotActivity(event);
  }

  // Periodic analysis (e.g., every 100 events)
  if (eventCount % 100 === 0) {
    var health = ecosystem.analyze();
    if (health.score < 50) {
      // Trigger alerts, adjust difficulty, accelerate retirement checks
      var optimal = difficulty.findOptimalDifficulty();
      var fleet = retirement.getStatus();
      console.log("Ecosystem degraded — optimal difficulty:", optimal.difficulty);
      console.log("Fleet:", fleet.active, "active,", fleet.probation, "on probation");
    }
  }
}

State Persistence

All five modules support exportState() and importState(state) for persistence across restarts. State is pure JSON — store it in your database, filesystem, or Redis.

// Save state
var states = {
  mimicry: mimicry.exportState(),
  attribution: attribution.exportState(),
  difficulty: difficulty.exportState(),
  ecosystem: ecosystem.exportState(),
  retirement: retirement.exportState()
};
fs.writeFileSync("intelligence-state.json", JSON.stringify(states));

// Restore state
var saved = JSON.parse(fs.readFileSync("intelligence-state.json"));
mimicry.importState(saved.mimicry);
attribution.importState(saved.attribution);
difficulty.importState(saved.difficulty);
ecosystem.importState(saved.ecosystem);
retirement.importState(saved.retirement);
