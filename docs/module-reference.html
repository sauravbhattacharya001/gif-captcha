Core

The foundational modules that power challenge creation, validation, and shared utilities.

index.js core

Main entry point. Re-exports the public API surface: challenge creation, set analysis, difficulty scoring, and configuration validation. Import this module for the standard gif-captcha interface.

const gifCaptcha = require('gif-captcha');

shared-utils.js core

Shared utility functions extracted from the core module (issue #91). Provides:

  • createChallenge() / pickChallenges() — challenge generation
  • validateAnswer() — answer verification with constant-time comparison
  • LruTracker — bounded LRU cache for session/rate tracking
  • secureRandomInt() — crypto-safe random number generation
  • createSanitizer() / sanitize() — input sanitization
  • loadGifWithRetry() — resilient GIF loading with exponential backoff
  • Statistical helpers: _mean, _median, _stddev, _percentile

crypto-utils.js core

Cryptographic primitives for CAPTCHA token generation and verification. Wraps Node.js crypto module with CAPTCHA-specific helpers for HMAC signing, token generation, and constant-time comparisons.

config-validator.js core

Validates and normalizes gif-captcha configuration objects. Ensures required fields are present, types are correct, and ranges are within bounds. Also exports createChallengeAnalytics() for challenge quality scoring.

const { createConfigValidator, createChallengeAnalytics } = require('./src/config-validator');

csv-utils.js core

CSV parsing and formatting utilities for data import/export across analytics modules. Handles quoting, escaping, and streaming for large datasets.

Security & Trust

Modules that detect bots, prevent abuse, and score user trustworthiness.

trust-score-engine.js security

Computes a composite trust score from behavioral signals, solve patterns, and session history. Uses bounded LRU tracking to prevent memory growth. Scores inform difficulty adjustment and bot flagging.

const { createTrustScoreEngine } = require('./src/trust-score-engine');

behavioral-biometrics.js security

Analyzes mouse movements, click patterns, and timing biometrics to distinguish human interaction from bot automation. Builds behavioral fingerprints without storing PII.

bot-signature-database.js security

Maintains a database of known bot behavioral signatures. Matches incoming solve patterns against known bot fingerprints for early detection.

const { createBotSignatureDatabase } = require('./src/bot-signature-database');

fraud-ring-detector.js security

Detects coordinated solving rings by analyzing cross-session patterns — shared IPs, solve timing correlations, and answer distribution anomalies.

const { createFraudRingDetector } = require('./src/fraud-ring-detector');

captcha-replay-detector.js security

Prevents replay attacks by tracking challenge tokens and detecting reuse. Uses bounded memory with LRU eviction (issue #127).

const { CaptchaReplayDetector } = require('./src/captcha-replay-detector');

honeypot-injector.js security

Injects invisible honeypot challenges that only bots would solve. Triggers on honeypot interaction provide high-confidence bot detection signals.

const { createHoneypotInjector } = require('./src/honeypot-injector');

solve-pattern-fingerprinter.js security

Builds behavioral fingerprints from solve patterns — timing distributions, error rates, difficulty progression — to identify returning users and bot networks without cookies.

const { createSolvePatternFingerprinter } = require('./src/solve-pattern-fingerprinter');

geo-risk-scorer.js security

Scores geographic risk based on IP geolocation. Uses O(1) hash-set lookups (replacing indexOf scans) for country risk classification. Feeds into trust scoring.

session-risk-aggregator.js security

Aggregates risk signals from multiple sources (trust engine, biometrics, geo, replay detection) into a unified session risk profile for real-time decision making.

captcha-rate-limiter.js security

Token-bucket rate limiting for CAPTCHA solve attempts. Prevents brute-force attacks while allowing legitimate retry patterns.

Analytics & Monitoring

Modules for measuring, analyzing, and reporting on CAPTCHA system health and user behavior.

captcha-stats-collector.js analytics

Collects and aggregates CAPTCHA performance metrics — solve rates, timing distributions, error rates. Exports percentile() for statistical analysis.

const { createStatsCollector, percentile } = require('./src/captcha-stats-collector');

captcha-anomaly-detector.js analytics

Detects anomalous patterns in CAPTCHA traffic — solve rate spikes, timing shifts, geographic anomalies — using statistical deviation analysis.

captcha-traffic-analyzer.js analytics

Analyzes CAPTCHA traffic patterns over time — request volumes, peak hours, geographic distribution, and trend detection.

const { createCaptchaTrafficAnalyzer } = require('./src/captcha-traffic-analyzer');

solve-funnel-analyzer.js analytics

Tracks the challenge solve funnel from presentation → attempt → success/failure. Identifies drop-off points with bounded memory (maxSessions/maxRecords LRU eviction).

const { createFunnelAnalyzer, STAGES } = require('./src/solve-funnel-analyzer');

response-time-profiler.js analytics

Profiles response times across the CAPTCHA pipeline — generation, presentation, validation — to identify bottlenecks.

const { createResponseTimeProfiler } = require('./src/response-time-profiler');

captcha-health-monitor.js analytics

Continuous health monitoring for the CAPTCHA system. Tracks uptime, error rates, latency percentiles, and triggers alerts when thresholds are exceeded.

captcha-fatigue-detector.js analytics

Detects user fatigue from repeated CAPTCHA challenges — declining solve rates, increasing abandon rates, frustration signals — and recommends difficulty reduction.

cognitive-fingerprint (HTML dashboard) analytics

Interactive dashboard for visualizing cognitive load patterns and user behavior clusters. Located at cognitive-fingerprint.html.

captcha-audit-log.js analytics

Structured audit logging for CAPTCHA events. Supports severity levels and event type validation for compliance requirements.

const { createAuditLog, VALID_EVENTS, SEVERITY } = require('./src/captcha-audit-log');

compliance-reporter.js analytics

Generates compliance reports for accessibility standards (WCAG), privacy regulations (GDPR), and security audits. Outputs structured reports suitable for stakeholder review.

const { createComplianceReporter } = require('./src/compliance-reporter');

Challenge Management

Modules that manage the lifecycle, difficulty, and diversity of CAPTCHA challenges.

adaptive-difficulty-tuner.js challenge

Dynamically adjusts challenge difficulty based on real-time solve rates and user trust scores. Maintains target solve rates while maximizing bot detection.

challenge-pool-manager.js challenge

Manages the pool of available challenges — adding, retiring, and selecting challenges based on freshness, difficulty distribution, and usage counts.

const { createChallengePoolManager } = require('./src/challenge-pool-manager');

challenge-rotation-scheduler.js challenge

Schedules periodic rotation of active challenge sets to prevent bot training on static challenge pools.

challenge-decay-manager.js challenge

Tracks challenge staleness and applies time-based decay to challenge effectiveness scores. Ensures old challenges are retired before bots can learn them.

const { createChallengeDecayManager } = require('./src/challenge-decay-manager');

challenge-diversity-analyzer.js challenge

Analyzes the diversity of challenge presentations — visual variety, answer distribution, difficulty spread — to prevent pattern exploitation.

challenge-template-engine.js challenge

Template engine for generating challenge variations from base templates. Supports parameterized visual transforms and answer mutations.

captcha-accessibility-analyzer.js challenge

Evaluates CAPTCHA challenges for accessibility — color contrast, animation speed, cognitive load, alternative modalities — against WCAG guidelines.

captcha-strength-scorer.js challenge

Scores the cryptographic and behavioral strength of individual challenges. Factors in entropy, answer space, visual complexity, and known attack resistance.
